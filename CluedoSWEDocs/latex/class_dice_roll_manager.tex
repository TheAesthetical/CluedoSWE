\doxysubsection{Detailed Description}
Attach to:\+ Roll Button (Canvas \texorpdfstring{$>$}{>} Background \texorpdfstring{$>$}{>} Options Panel \texorpdfstring{$>$}{>} Roll Button). 

No Game\+Manager needed. Completely self-\/contained.

Rolls 2 dice (each 1-\/6) giving a total of 2-\/12 moves. Shows a tumbling animation before settling on the final result. The Roll button is disabled after rolling and re-\/enabled when \doxylink{class_player_turn_manager}{Player\+Turn\+Manager} advances to the next turn.

SETUP:\+
\begin{DoxyEnumerate}
\item Select "{}\+Roll Button"{} in the Hierarchy.
\item Add Component → \doxylink{class_dice_roll_manager}{Dice\+Roll\+Manager}.
\item Create a small "{}\+Dice Panel"{} somewhere on the Canvas to display results:\+ • Two Image Game\+Objects for the die faces (Die 1, Die 2)
\begin{DoxyItemize}
\item Optionally assign 6 die-\/face sprites (index 0 = face 1, index 5 = face 6)
\item Or leave die\+Sprites empty:\+ each Image needs a child TMP Text and numbers will be shown as text instead • One Text\+Mesh\+Pro\+UGUI showing the total, e.\+g. "{}\+Moves:\+ 9"{}
\end{DoxyItemize}
\item Drag those objects into the slots below.
\item Drag "{}\+End Turn Button"{} into the End Turn Button slot so this script knows when to re-\/enable rolling for the next turn.
\end{DoxyEnumerate}

NOTE:\+ End Turn button click is shared between this script and \doxylink{class_player_turn_manager}{Player\+Turn\+Manager} — both listen to the same button independently, so there is no conflict.
